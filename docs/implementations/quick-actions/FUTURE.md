# Future Notes

## Quick Actions Direction

- Prioritize native home-screen quick actions first, then reuse the same action model for in-app home actions in a later phase.
- Keep `New Story` as a pinned, non-removable action.
- Add `Templates` (open all templates) as an optional pinned action candidate.
- Use a `Pinned` treatment instead of disabled remove buttons.
- Keep reorder constrained so pinned actions stay in the pinned zone.

## Home Quick Actions UX Follow-ups

- Keep one unified available-actions section so default actions and picker actions share the same visual parent.
- Preserve animated insert/remove behavior for default actions.
- For in-app home customization, launch as a separate setting after native quick actions prove stable.
